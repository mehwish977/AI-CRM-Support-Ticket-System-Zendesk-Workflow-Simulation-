Tagging System

In this project, tags are used to label tickets in a consistent and organized way. Instead of adding random tags, a fixed structure is followed so that tickets can be easily grouped and understood.

Main tags used

order_delay  
refund_request  
technical_issue  
general_inquiry  
complaint  

Additional tags

delivery_issue  
product_mismatch  
login_issue  
website_bug  
shipping_query  

How tags are used

Each ticket includes at least one main tag that represents the overall issue. Additional tags are used to give more detail about the specific problem.

For example, a refund request may also include a tag like product_mismatch to explain the reason behind the request.

Why this matters

Using a consistent tagging system makes it easier to:
- filter tickets  
- identify common issues  
- analyze patterns in customer problems  

It also helps maintain clarity when working with a large number of tickets.

Final note

Tags should be simple, clear, and consistent. Avoid creating unnecessary or duplicate tags, as that can make the system harder to manage.
